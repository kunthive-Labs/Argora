What it is

A prospecting tool, not just a scraper.

The premise is simple: a local business with reviews, a rating, and a phone number — but no website — is a warm lead for anyone who sells websites. Argora drives Google Maps with a Playwright scraper, pulls raw places, then runs them through a lead-intelligence layer that splits the no-website prospects from their competitors and ranks them by reviews × rating — so established businesses that simply never went online surface first.

The split is deliberate. The scraper is a maintenance treadmill — selectors rot, IPs throttle — so it stays thin and isolated. The analysis layer is the real moat: it's column-normalized, so it ingests output from any engine unchanged, and it never breaks when Google changes its markup. You get CSVs to call from and idempotent Supabase SQL to push into your CRM.

Highlights

What it actually does.

Every capability below ships in the repo today.

No-website detection

Splits every place into a website_status — flags the businesses with reviews and a phone but no site as your leads.

Reviews × rating ranking

Leads are ordered so the most-established, least-online businesses rise to the top of your call list.

Competitor lists

Pulls the same-category businesses that do have a site — pitch ammo: "your neighbour has a site and N reviews."

Sector presets

Driving-school, real-estate, gym, dentist… defined as a dict in sectors.py. Add one by copying a block.

Idempotent Supabase SQL

Turns a LEADS csv into an INSERT that can never add a row twice — guarded four ways, verified against Postgres 16.

Live scrape, in your browser

Run python app.py for a one-file UI that streams every place over SSE with running lead and competitor counts.

The pipeline

Scrape → split → rank → ship.

Four stages, deliberately separated so the fragile part never poisons the moat.

01 / SCRAPE

Drive Maps

A Playwright/Chromium scraper (scraper.py) pulls raw places for a sector in an area — logged out, headed, gentle.

02 / ANALYZE

Split & normalize

analyze.py normalizes columns and sorts every place into LEADS, COMPETITORS, and ALL.

03 / RANK

Order the calls

No-website leads with a phone are ranked by reviews × rating — who you call, top-down.

04 / SHIP

CSV + SQL

sql_gen.py turns a LEADS csv into idempotent Supabase SQL for the KunthiveOS leads table.

Get started

Up and scraping in four lines.

One-time setup, then the one command. These are the real commands from the README.

argora — setup & run 
# 1. one-time setup
$ python3 -m venv .venv && source .venv/bin/activate
$ pip install -r requirements.txt
$ playwright install chromium

# 2. the one command — opens the web app in your browser
$ python app.py

# …or the scriptable CLI: a sector + an area
$ python run.py driving-school "Jayanagar, Bengaluru"
$ python run.py real-estate "Whitefield, Bengaluru" --max 120

# 3. generate idempotent Supabase SQL from a LEADS csv
$ python -m leadfinder.sql_gen data/leads/gym-yelahanka-LEADS.csv \
      --dataset argora/gym-yelahanka --out data/sql/gym-yelahanka.sql

Before you scale, read BARRIERS.md — Google caps ~120 results per search, so scrape area-by-area, stay logged out, and keep runs gentle. No CAPTCHA-evasion tooling.
